    最优化理论知识整理

    开篇：优化问题到底在解决什么？

    想象你是一家工厂的老板，有限的原材料，有限的工人，有限的机器。你要决定生产多少A产品、多少B产品，才能让利润最大。这就是一个优化问题。

    或者你是一个登山者，站在云雾缭绕的山上，想找到最低的谷底。你看不到全貌，只能感受脚下的坡度。这也是优化问题。

    优化的本质：在约束条件下，找到让目标函数达到最优（最大或最小）的那组变量值。

    数学语言翻译一下：
    $$\min f(x) \quad \text{subject to} \quad g_i(x) \le 0, , h_j(x) = 0$$

    现在，让我们从最简单的情况开始，一层层揭开优化理论的面纱。

    第一幕：线性的世界

    场景一：两个变量，一张图纸

    问题：某工厂生产桌子和椅子。一张桌子利润3元，一把椅子利润2元。车间限制：木料够做4套家具，工时够做5件大型家具（桌子算2件）。问怎么安排生产？

    数学化：
    $$\max z = 3x_1 + 2x_2$$
    $$\text{s.t.} \begin{cases} x_1 + x_2 \le 4 \ 2x_1 + x_2 \le 5 \ x_1, x_2 \ge 0 \end{cases}$$

    图解法：拿出方格纸。

    第一步：画出可行域

    • $x_1 + x_2 = 4$ 是一条直线，可行域在它下方
    • $2x_1 + x_2 = 5$ 是另一条直线，可行域在它下方
    • 再加上 $x_1 \ge 0, x_2 \ge 0$

    四条线围成一个四边形，顶点是：$(0,0)$，$(0,4)$，$(1,3)$，$(2.5,0)$

    第二步：目标函数 $z = 3x_1 + 2x_2$ 改写成 $x_2 = -\frac{3}{2}x_1 + \frac{z}{2}$

    这是一族平行线，斜率固定为 $-3/2$，截距 $z/2$ 越大越好。

    第三步：平移这条直线，让它尽量”往上顶”，同时还能碰到可行域。最后一个碰到的点就是最优解。

    画图可知：最后接触点是 $(1,3)$，利润 $z = 3 \times 1 + 2 \times 3 = 9$ 元。

    核心洞察：线性规划的最优解一定在可行域的顶点上。因为目标函数是平面，可行域是多边形，平面在多边形上”滑动”，最后必定卡在某个角上。

    场景二：单纯形法——沿着棱边爬山

    图解法只能处理两个变量。三个变量呢？十个？一百个？

    这时候需要 单纯形法：一个顶点一个顶点地跳，每次跳到相邻的更优顶点，直到无路可跳。

    标准形式（引入松弛变量把不等式变等式）：
    $$\min c^T x \quad \text{s.t.} \quad Ax = b, , x \ge 0$$

    核心概念：

    • 基变量：$m$ 个”主角”变量，对应单位矩阵的列
    • 非基变量：其余”群众演员”，全设为 0
    • 基可行解：一个顶点

    单纯形表的故事：

    基变量 $x_1$ $x_2$ $x_3$ $x_4$ RHS
    $x_3$ 1 1 1 0 6
    $x_4$ 2 1 0 1 8
    $z$ -1 -2 0 0 0

    最下面一行是”检验数”。负数意味着”这个方向还能下降”。

    检验数最负的是 $x_2$（$-2$），所以让 $x_2$ 进基。

    用最小比值规则：$\min{6/1, 8/1} = 6$，所以 $x_3$ 出基。

    更新单纯形表，继续迭代，直到检验数全部 $\ge 0$，大功告成。

    直观理解：你站在多面体的一个角上，环顾四周的边，选最陡的那条往下走。走到下一个角，再选，再走。直到四周都是上坡路，你就到了谷底。

    例题快速过：

    $$\min -x_1 - 2x_2 \quad \text{s.t.} \quad x_1 + x_2 \le 6, , 2x_1 + x_2 \le 8, , x_1, x_2 \ge 0$$

    引入松弛变量 $x_3, x_4$。初始顶点 $(0,0)$，目标值 $0$。

    第一轮：$x_2$ 进基（检验数 $-2$），$x_3$ 出基，到达 $(0,6)$，$z = -12$

    第二轮：$x_1$ 进基，$x_4$ 出基，到达 $(2,4)$，$z = -10$

    检验数全非负，停止。最优解 $(2,4)$。

    场景三：对偶理论——从另一个角度看世界

    每个线性规划都有一个”镜像问题”，叫对偶问题。

    原问题（Primal）：工厂老板视角，最小化成本
    $$\min c^T x \quad \text{s.t.} \quad Ax \ge b, , x \ge 0$$

    对偶问题（Dual）：原料商视角，最大化收益
    $$\max b^T y \quad \text{s.t.} \quad A^T y \le c, , y \ge 0$$

    弱对偶定理：对偶的任何可行解都是原问题最优值的界。

    强对偶定理：如果原问题有最优解，对偶也有，且最优值相等。

    互补松弛：在最优点，如果原问题某约束松弛（不紧），对偶变量为 0；反之，对偶变量为正，原约束必紧。

    经济学解释：

    原问题：我有这些资源，怎么用最便宜？

    对偶问题：如果我把资源卖给你，定什么价你才愿意买（且我赚最多）？

    最优点：资源的影子价格。

    例题：

    原：$\min 2x_1 + 3x_2$ s.t. $x_1 + x_2 \ge 4$, $2x_1 + x_2 \ge 5$, $x_1, x_2 \ge 0$

    对偶：$\max 4y_1 + 5y_2$ s.t. $y_1 + 2y_2 \le 2$, $y_1 + y_2 \le 3$, $y_1, y_2 \ge 0$

    求解原问题得 $x^* = (1, 3)$，$z = 11$

    求解对偶得 $y^* = (2, 0)$（或其他组合），对偶最优值也是 $11$。验证强对偶定理成立。

    第二幕：无约束非线性优化——在山地里找谷底

    离开线性的安全区，进入非线性的世界。现在没有”顶点必最优”的保证，我们需要新的武器。

    问题形式：
    $$\min f(x), \quad x \in \mathbb{R}^n$$

    没有约束，自由探索。但函数可能很复杂——有多个波峰波谷，有狭长山谷，有陡峭悬崖。

    迭代框架：从某个初始点 $x_0$ 出发，重复以下步骤：

    1. 选择一个搜索方向 $d_k$
    2. 沿该方向确定步长 $\alpha_k$
    3. 更新：$x_{k+1} = x_k + \alpha_k d_k$
    4. 检查是否收敛，否则回到步骤 1

    关键问题有两个：

    • 如何选方向？ → 这决定了算法的智能程度
    • 如何定步长？ → 这决定了每步走多远

    让我们先解决第二个问题。

    插曲：一维搜索——在直线上找最低点

    确定了方向 $d_k$ 后，问题变成：沿着射线 $x_k + \alpha d_k$ 找最优的 $\alpha$。

    这是一维优化问题：
    $$\min_{\alpha \ge 0} \phi(\alpha) = f(x_k + \alpha d_k)$$

    虽然看起来简单，但这是所有迭代算法的”发动机”。我们需要一套工具箱。

    武器一：黄金分割法——不用导数的智慧

    思路：在区间里插两个点，比较函数值，舍弃较大的那半，缩小区间。

    黄金比例：$\tau = 0.618$

    每次在 $[a, b]$ 中取 $\lambda = a + 0.382(b-a)$，$\mu = a + 0.618(b-a)$

    比较 $\phi(\lambda)$ 和 $\phi(\mu)$：

    • 若 $\phi(\lambda) < \phi(\mu)$，新区间 $[a, \mu]$
    • 否则 $[\lambda, b]$

    妙处在于：新区间的边界点可以复用旧区间的内点，只需算一个新函数值。

    例题：求 $\phi(\alpha) = (\alpha - 2)^2$ 在 $[0, 4]$ 上的最小值

    初始 $[0, 4]$：$\lambda = 1.528$，$\mu = 2.472$

    $\phi(1.528) = 0.223$，$\phi(2.472) = 0.223$（接近）

    缩小区间继续，几轮后收敛到 $\alpha = 2$，$\phi = 0$。

    适用场景：不可导函数，或导数难算。收敛慢但稳健。

    武器二：牛顿法（一维）——二次近似的暴力

    思路：用二次函数 $q(\alpha) = \phi(\alpha_k) + \phi’(\alpha_k)(\alpha - \alpha_k) + \frac{1}{2}\phi’’(\alpha_k)(\alpha - \alpha_k)^2$ 近似 $\phi$

    二次函数的极值点：$\alpha^* = \alpha_k - \frac{\phi’(\alpha_k)}{\phi’’(\alpha_k)}$

    如果 $\phi$ 真是二次的，一步到位。否则迭代几次。

    例题：$\phi(\alpha) = \alpha^3 - 3\alpha$，初始 $\alpha_0 = 2$

    $\phi’(\alpha) = 3\alpha^2 - 3$，$\phi’’(\alpha) = 6\alpha$

    $\alpha_1 = 2 - \frac{3 \times 4 - 3}{6 \times 2} = 2 - \frac{9}{12} = 1.25$

    $\alpha_2 = 1.25 - \frac{3 \times 1.5625 - 3}{6 \times 1.25} = 1.04$

    几步收敛到 $\alpha^* = 1$，$\phi(1) = -2$

    优点：收敛超快（二次收敛）

    缺点：需要二阶导，初值不好可能飞

    武器三：割线法——穷人的牛顿法

    思路：没有二阶导？用差分近似！

    $$\alpha_{k+1} = \alpha_k - \phi’(\alpha_k) \frac{\alpha_k - \alpha_{k-1}}{\phi’(\alpha_k) - \phi’(\alpha_{k-1})}$$

    用前两点的导数值构造割线，割线与横轴交点就是新的估计。

    例题：$\phi(\alpha) = \alpha^2 - 4\alpha + 3$，初始 $\alpha_0 = 0, \alpha_1 = 1$

    $\phi’(\alpha) = 2\alpha - 4$

    $\phi’(0) = -4$，$\phi’(1) = -2$

    $\alpha_2 = 1 - (-2) \frac{1 - 0}{-2 - (-4)} = 1 + 1 = 2$

    $\phi’(2) = 0$，找到极值点！

    优点：不需要二阶导，收敛也不错

    缺点：需要两个初始点

    武器四：抛物线法——三点定一抛物线

    思路：用三个点 $(α_1, \phi_1), (α_2, \phi_2), (α_3, \phi_3)$ 拟合抛物线，抛物线的顶点就是新估计。

    拟合公式复杂，但思路简单：三点确定二次曲线，二次曲线的极值点显式可解。

    例题：$\phi(\alpha) = \alpha^2 - 4\alpha + 5$，三点 $(0,5), (2,1), (4,5)$

    显然对称，顶点在中间 $\alpha = 2$。

    实际用公式算也是 $\alpha^* = 2$。

    适用：光滑函数，不想算导数。

    小结：一维搜索工具箱准备好了。黄金分割最稳，牛顿法最快，割线法折中，抛物线法简便。接下来，我们要用它们来为多维优化”定步长”。

    策略一：最速下降法——最直接的下坡

    核心：梯度的负方向是函数下降最快的方向，那就沿着它走。

    $$x_{k+1} = x_k - \alpha_k \nabla f(x_k)$$

    $\alpha_k$ 用一维搜索确定（精确线搜索）或固定步长。

    例题：$f(x_1, x_2) = x_1^2 + 4x_2^2$，初始 $(2, 2)$

    $\nabla f = (2x_1, 8x_2)$

    第 1 步：$\nabla f(2, 2) = (4, 16)$，方向 $d_0 = -(4, 16)$

    沿方向最小化：$\phi(\alpha) = (2 - 4\alpha)^2 + 4(2 - 16\alpha)^2$

    用牛顿法或直接求导：$\phi’(\alpha) = -8(2-4\alpha) - 128(2-16\alpha) = 0$

    解得 $\alpha \approx 0.265$

    $x_1 = (2,2) - 0.265(4,16) = (0.94, -2.24)$

    第 2 步：$\nabla f(0.94, -2.24) = (1.88, -17.92)$，方向 $d_1 = -(1.88, -17.92)$

    用黄金分割或牛顿法定步长，继续…

    轨迹呈现”之字形”，因为每次新方向和上次方向正交，在椭圆等高线上反复横跳。

    问题：收敛慢，尤其是细长椭圆（Hessian 条件数大）时，可能需要上千步。

    适用：简单问题，或作为其他算法的备选方向。

    策略二：共轭梯度法——不走回头路

    核心思想：最速下降的问题是”破坏旧成果”。共轭梯度法构造一组特殊方向，在这些方向上的搜索相互不干扰。

    共轭性：对于二次函数 $f(x) = \frac{1}{2}x^T A x - b^T x$，方向 $d_i$ 和 $d_j$ 共轭是指 $d_i^T A d_j = 0$。

    Fletcher-Reeves 公式：
    $$d_0 = -\nabla f(x_0)$$
    $$d_{k+1} = -\nabla f(x_{k+1}) + \beta_k d_k$$
    $$\beta_k = \frac{|\nabla f(x_{k+1})|^2}{|\nabla f(x_k)|^2}$$

    $\beta_k$ 是”记忆系数”，让新方向带上旧方向的信息。

    理论保证：对于 $n$ 维二次函数，最多 $n$ 步精确收敛。

    例题：$f(x_1, x_2) = x_1^2 + 4x_2^2$，初始 $(2, 2)$

    第 1 步：同最速下降，沿 $d_0 = -(4, 16)$ 精确线搜索到达 $(0.94, -2.24)$

    第 2 步：$\nabla f_1 = (1.88, -17.28)$

    $\beta_0 = \frac{1.88^2 + 17.28^2}{4^2 + 16^2} = \frac{302.2}{272} = 1.11$

    $d_1 = -(1.88, -17.28) + 1.11 \times -(4, 16) = (-6.32, 0.52)$（方向改变！）

    沿 $d_1$ 精确线搜索（用牛顿法或黄金分割），直接到达 $(0, 0)$！

    两步收敛，比最速下降的上千步快太多。

    适用：中大规模二次或近似二次问题。非二次函数需要周期重启。

    策略三：牛顿法——用曲率信息加速

    核心：不仅用一阶信息（梯度），还用二阶信息（Hessian）。在当前点做二次近似，直接跳到近似函数的极值点。

    迭代公式：
    $$x_{k+1} = x_k - H_k^{-1} \nabla f(x_k)$$

    其中 $H_k = \nabla^2 f(x_k)$ 是 Hessian 矩阵。

    注意这里的步长隐含为 1（纯牛顿方向），也可以结合一维搜索：$x_{k+1} = x_k - \alpha_k H_k^{-1} \nabla f(x_k)$。

    直观理解：梯度告诉你”坡度”，Hessian 告诉你”坡度的变化率”（曲率）。如果知道曲率，可以更准确地估计谷底位置。

    例题：$f(x_1, x_2) = x_1^2 + 2x_2^2 - 2x_1 - 4x_2 + 3$

    $\nabla f = (2x_1 - 2, 4x_2 - 4)$

    $H = \begin{pmatrix} 2 & 0 \ 0 & 4 \end{pmatrix}$

    初始 $(0, 0)$：

    $\nabla f(0,0) = (-2, -4)$

    $x_1 = (0,0) - \begin{pmatrix} 2 & 0 \ 0 & 4 \end{pmatrix}^{-1} \begin{pmatrix} -2 \ -4 \end{pmatrix} = (0,0) + \begin{pmatrix} 1 \ 1 \end{pmatrix} = (1, 1)$

    检验：$\nabla f(1,1) = (0, 0)$。一步到位！

    对于非二次函数，比如 Rosenbrock 函数 $f = 100(x_2 - x_1^2)^2 + (1-x_1)^2$：

    初始 $(-1.2, 1)$，计算 Hessian，每步可能需要用黄金分割或回溯线搜索确定步长，迭代几步即可收敛到 $(1, 1)$。

    优点：收敛极快（二次收敛），对二次函数一步到位

    缺点：

    1. 计算 Hessian 代价高（$O(n^2)$ 存储，$O(n^3)$ 求逆）
    2. Hessian 可能不正定，导致方向不是下降方向
    3. 初始点不好可能发散

    策略四：拟牛顿法——穷人的牛顿法（真正实用版）

    核心：牛顿法太贵，能不能用一阶信息（梯度）逐步”猜”出 Hessian 的逆？

    答案是可以。这就是拟牛顿法。

    BFGS 算法（最流行的拟牛顿法）：

    维护一个矩阵 $B_k \approx H_k^{-1}$，每步更新：

    $$s_k = x_{k+1} - x_k \quad (\text{位置变化})$$
    $$y_k = \nabla f(x_{k+1}) - \nabla f(x_k) \quad (\text{梯度变化})$$

    $$B_{k+1} = B_k + \frac{y_k y_k^T}{y_k^T s_k} - \frac{B_k s_k s_k^T B_k}{s_k^T B_k s_k}$$

    初始 $B_0 = I$（单位矩阵）。

    搜索方向：$d_k = -B_k \nabla f(x_k)$

    步长：用回溯线搜索或 Wolfe 条件确定 $\alpha_k$（这里就用到一维搜索工具箱）。

    直观理解：每走一步，观察”走了多远”（$s_k$）和”坡度变了多少”（$y_k$），用这个信息调整对”地形曲率”的估计。就像盲人摸象，摸的信息越多，对大象的轮廓理解越准。

    例题：Rosenbrock 函数（香蕉函数）

    $$f(x_1, x_2) = 100(x_2 - x_1^2)^2 + (1-x_1)^2$$

    这是个非常”难缠”的函数，等高线像香蕉，狭长弯曲。

    初始点 $(-1.2, 1)$，$B_0 = I$

    第 1 步：$d_0 = -B_0 \nabla f = -\nabla f = (215.6, 88)^T$（方向）

    用黄金分割或 Wolfe 线搜索得 $\alpha_0 \approx 0.001$（很小，因为函数陡峭）

    $x_1 = (-1.2, 1) + 0.001(215.6, 88) = (-0.984, 1.088)$

    计算 $s_0 = x_1 - x_0$，$y_0 = \nabla f(x_1) - \nabla f(x_0)$

    用 BFGS 公式更新 $B_1$

    第 2 步：用新的 $B_1$ 计算方向 $d_1 = -B_1 \nabla f(x_1)$（方向已经不是纯梯度了）

    继续线搜索，更新…

    重复 10-30 步，收敛到 $(1, 1)$（全局最优点）

    对比：最速下降需要几千步，甚至不收敛；牛顿法可能因 Hessian 病态失败；BFGS 稳健且快速。

    优点：

    • 不需要 Hessian，只需梯度
    • 收敛快（超线性收敛）
    • 实践中非常可靠

    缺点：

    • 需要存储 $n \times n$ 矩阵（但有 L-BFGS 等低内存版本）

    工业界地位：BFGS 和 L-BFGS 是无约束优化的”默认选择”，几乎所有优化库都实现了它们。

    第二幕小结：

    无约束优化的迭代框架是：选方向 + 定步长。

    • 方向策略：从简单的梯度，到聪明的共轭方向，到利用曲率的牛顿方向，到近似曲率的拟牛顿方向
    • 步长策略：一维搜索工具箱（黄金分割、牛顿法、割线法、抛物线法）

    两者结合，就能在复杂地形中找到谷底。

    第三幕：约束优化理论——在墙内找谷底

    前面假设你可以自由行走。现在加上限制：不能越界，必须在某个区域内。

    理论基石：Kuhn-Tucker (KT) 必要条件

    问题：
    $$\min f(x) \quad \text{s.t.} \quad g_i(x) \le 0 , (i=1,\dots,m), \quad h_j(x) = 0 , (j=1,\dots,p)$$

    在最优点 $x^*$ 必须满足：

    1. 梯度平衡：
      $$\nabla f(x^) + \sum_{i=1}^m \lambda_i^ \nabla g_i(x^) + \sum_{j=1}^p \mu_j^ \nabla h_j(x^*) = 0$$

    2. 互补松弛：
      $$\lambda_i^* g_i(x^*) = 0, \quad i=1,\dots,m$$

    3. 原始可行：
      $$g_i(x^) \le 0, \quad h_j(x^) = 0$$

    4. 对偶可行：
      $$\lambda_i^* \ge 0$$

    物理直觉：

    想象你在山谷里，但周围有几堵墙（约束）。

    • 如果你在空地中间，那就是无约束问题，梯度为零即可
    • 如果你被某堵墙挡住了，你想下山（$\nabla f$），但墙的”推力”（$\lambda \nabla g$）正好抵消你的”下山冲动”
    • 互补松弛：如果墙离你很远（$g_i < 0$），它不影响你（$\lambda_i = 0$）；只有紧贴墙时（$g_i = 0$），墙才有推力（$\lambda_i > 0$）

    例题：
    $$\min f(x_1, x_2) = x_1^2 + x_2^2 \quad \text{s.t.} \quad x_1 + x_2 \ge 1$$

    标准化：$g(x) = 1 - x_1 - x_2 \le 0$

    拉格朗日函数：$L = x_1^2 + x_2^2 + \lambda(1 - x_1 - x_2)$

    KT 条件：
    $$\frac{\partial L}{\partial x_1} = 2x_1 - \lambda = 0 \Rightarrow x_1 = \lambda/2$$
    $$\frac{\partial L}{\partial x_2} = 2x_2 - \lambda = 0 \Rightarrow x_2 = \lambda/2$$
    $$\lambda(1 - x_1 - x_2) = 0$$
    $$1 - x_1 - x_2 \le 0, \quad \lambda \ge 0$$

    情况 1：$\lambda = 0$

    则 $x_1 = x_2 = 0$，但 $1 - 0 - 0 = 1 > 0$，违反约束。舍弃。

    情况 2：$1 - x_1 - x_2 = 0$（约束紧）

    代入：$x_1 = x_2 = \lambda/2$，则 $1 - \lambda/2 - \lambda/2 = 0 \Rightarrow \lambda = 1$

    所以 $x_1 = x_2 = 0.5$

    检验：$f(0.5, 0.5) = 0.5$

    几何意义：原点到直线 $x_1 + x_2 = 1$ 的最短距离，投影点就是 $(0.5, 0.5)$。

    梯度 $\nabla f = (1, 1)$ 指向远离原点，约束梯度 $\nabla g = (-1, -1)$ 指向可行域内，两者反向，用乘子 $\lambda = 1$ 平衡。

    第四幕：约束优化算法——实战武器库

    有了 KT 条件的理论指导，我们需要算法来实际求解。这些算法同样需要在每步确定方向和步长，但要考虑约束的影响。

    武器一：可行方向法（Zoutendijk）

    思路：从可行域内部出发，每次找一个既下降又可行的方向。

    可行下降方向 $d$ 满足：

    1. $\nabla f(x_k)^T d < 0$（下降）
    2. $\nabla g_i(x_k)^T d < 0$（离开约束边界，保持可行，对于紧约束 $i \in I$）

    Zoutendijk 方向子问题：
    $$\max \theta \quad \text{s.t.} \quad \nabla f^T d \le -\theta, , \nabla g_i^T d \le -\theta , (i \in I), , |d| \le 1$$

    其中 $I$ 是紧约束集合（$g_i(x_k) = 0$）。

    确定方向后，用一维搜索（黄金分割或回溯）确定步长，但要保证不越界。

    直观：在所有”不撞墙”的方向里，选下降最陡的。

    例题：
    $$\min x_1^2 + x_2^2 \quad \text{s.t.} \quad x_1 + x_2 \ge 2$$

    当前点 $x_k = (1.5, 0.4)$，约束 $g = 2 - x_1 - x_2 = 0.1 > 0$，内部点，约束不紧。

    直接用 $d = -\nabla f = -(3, 0.8)$（归一化后）

    用黄金分割沿此方向搜索，直到触及边界或到达最优。

    当前点 $(1, 1)$（边界上），$\nabla f = (2, 2)$，$\nabla g = (-1, -1)$

    子问题：$\max \theta$ s.t. $(2,2)^T d \le -\theta$, $(-1,-1)^T d \le -\theta$, $|d| \le 1$

    注意到 $(2,2)$ 和 $(-1,-1)$ 方向相反，解得 $\theta \to 0$，说明已到最优点附近。

    武器二：梯度投影法（Rosen）

    思路：把负梯度投影到约束的切空间上，得到既下降又不穿墙的方向。

    投影公式（线性等式约束 $Ax = b$）：
    $$d = P(-\nabla f)$$
    $$P = I - A^T(AA^T)^{-1}A$$

    确定方向后，用抛物线法或牛顿法做一维搜索。

    直观：你想下山（$-\nabla f$），但前方是悬崖（约束边界）。投影就是把”下山向量”分解成”沿着悬崖走”和”跳下悬崖”，我们只取”沿悬崖走”的分量。

    例题：
    $$\min (x_1 - 2)^2 + (x_2 - 1)^2 \quad \text{s.t.} \quad x_1 + x_2 = 2$$

    当前点 $(1, 1)$，$\nabla f = (-2, 0)$

    约束法向量 $A = (1, 1)$

    投影矩阵：
    $$P = I - \frac{1}{2} \begin{pmatrix} 1 \ 1 \end{pmatrix} (1, 1) = \begin{pmatrix} 0.5 & -0.5 \ -0.5 & 0.5 \end{pmatrix}$$

    投影梯度：
    $$d = P \begin{pmatrix} 2 \ 0 \end{pmatrix} = \begin{pmatrix} 1 \ -1 \end{pmatrix}$$

    沿 $(1, -1)$ 方向，用割线法或黄金分割确定步长，保持 $x_1 + x_2 = 2$，到达 $(1.5, 0.5)$

    继续迭代到 $(2, 0)$（真正的最优点）

    武器三：外点罚函数法——允许越界，但要付出代价

    思路：把约束项加到目标函数里，违反约束就罚你。罚得越重，越不敢违反。

    罚函数：
    $$P(x, \rho) = f(x) + \rho \sum_{i=1}^m [\max(0, g_i(x))]^2 + \rho \sum_{j=1}^p [h_j(x)]^2$$

    算法：

    1. 初始点 $x_0$（可以不可行），$\rho_0$ 较小
    2. 用 BFGS 或共轭梯度无约束最小化 $P(x, \rho_k)$ 得 $x_k$（内层用一维搜索）
    3. 增大 $\rho_{k+1} = 10 \rho_k$
    4. 重复直到收敛

    直观：可行域外围设电网，一开始电压低，可以越界探索；后来电压升高，被电回可行域。

    例题：
    $$\min x^2 \quad \text{s.t.} \quad x \ge 1$$

    $g(x) = 1 - x \le 0$

    罚函数：$P(x, \rho) = x^2 + \rho [\max(0, 1-x)]^2$

    $\rho = 1$：
    $$P(x,1) = x^2 + (1-x)^2 \quad (x < 1)$$

    用牛顿法或求导：$P’(x) = 2x - 2(1-x) = 4x - 2 = 0 \Rightarrow x = 0.5$

    违反约束，但罚得轻。

    $\rho = 10$：
    $$P(x,10) = x^2 + 10(1-x)^2$$
    $$P’(x) = 2x - 20(1-x) = 22x - 20 = 0 \Rightarrow x = 0.909$$

    接近边界。

    $\rho = 100$：用割线法或黄金分割优化，得 $x \approx 0.99$

    $\rho \to \infty$：$x \to 1$（最优解）

    优缺点：

    • 优点：简单，易实现，不需要可行初始点
    • 缺点：$\rho$ 太大导致 Hessian 病态，一维搜索困难；收敛慢

    武器四：内点障碍函数法——内部逼近

    思路：在可行域内部构造”势垒”，越靠近边界推力越大，随着势垒降低逼近边界。

    障碍函数（对数障碍）：
    $$B(x, \mu) = f(x) - \mu \sum_{i=1}^m \ln(-g_i(x))$$

    注意：$g_i(x) < 0$（严格可行），否则对数无定义。

    算法：

    1. 严格可行初始点 $x_0$，$\mu_0$ 较大
    2. 用牛顿法或 BFGS 无约束优化 $\min B(x, \mu_k)$ 得 $x_k$（内层用抛物线法或牛顿法做一维搜索）
    3. 减小 $\mu_{k+1} = \mu_k / 10$
    4. 重复直到收敛

    直观：可行域边界是”软墙”，靠近时有无形的反推力。一开始墙很硬，被挡在中间；后来墙变软，可以贴近边界。

    例题：
    $$\min (x-3)^2 \quad \text{s.t.} \quad x \le 2$$

    $g(x) = x - 2 \le 0$

    障碍函数：$B(x, \mu) = (x-3)^2 - \mu \ln(2-x)$（要求 $x < 2$）

    $\mu = 1$：
    $$B’(x) = 2(x-3) + \frac{1}{2-x} = 0$$

    用牛顿法迭代：$(x-3)(2-x) = -0.5$ → $x \approx 1.634$

    $\mu = 0.1$：
    $$(x-3)(2-x) = -0.05$$

    用割线法：$x \approx 1.926$

    $\mu \to 0$：用黄金分割精细搜索，$x \to 2$（最优解）

    优缺点：

    • 优点：迭代点始终可行，适合必须保持可行的问题
    • 缺点：需要严格可行初始点（有时难找）；靠近边界时一维搜索数值不稳定

    尾声：知识地图与复习锦囊

    从线性到非线性，从无约束到约束

    线性规划
├─ 图解法（几何直觉：顶点最优）
├─ 单纯形法（沿棱边跳顶点）
└─ 对偶理论（影子价格，经济学视角）

无约束非线性优化
├─ 迭代框架：选方向 + 定步长
├─ 一维搜索工具箱（为步长服务）
│  ├─ 黄金分割（不用导数，稳健）
│  ├─ 牛顿法（二次收敛，需二阶导）
│  ├─ 割线法（近似牛顿，只需一阶导）
│  └─ 抛物线法（三点拟合）
├─ 方向策略
│  ├─ 最速下降（简单但慢，之字形震荡）
│  ├─ 共轭梯度（利用历史，n 步收敛）
│  ├─ 牛顿法（二次收敛，计算贵）
│  └─ 拟牛顿（BFGS 实用之王）

约束非线性优化
├─ 理论：KT 条件（梯度平衡 + 互补松弛）
├─ 可行方向法（Zoutendijk 选方向 + 一维搜索定步长）
├─ 梯度投影法（Rosen 投影到切空间 + 一维搜索）
├─ 罚函数法（外部逼近，内层无约束优化 + 一维搜索）
└─ 障碍函数法（内部逼近，内层无约束优化 + 一维搜索）

    关键洞察：一维搜索是所有迭代算法的”发动机”，无论无约束还是约束，都需要它来确定步长。

    复习秘籍：理解 > 公式 > 例题

    第一遍：只看”直观理解”

    • 凸函数：碗，局部=全局
    • 单纯形：走顶点，选最陡边
    • 对偶：原料商与工厂老板的博弈
    • 黄金分割：舍弃较高那半
    • 最速下降：贪心下坡，但会震荡
    • 共轭梯度：不破坏旧成果
    • 牛顿法：用曲率信息，抛物线顶点
    • BFGS：盲人摸象，逐步猜 Hessian
    • KT 条件：下山冲动被墙挡住
    • 罚函数：电网，罚越重越不敢越界
    • 障碍函数：软墙，靠近时反推

    第二遍：看”例题”

    • 跟着例题手算一遍，感受算法的运作
    • 不需要记住每个数字，记住”步骤”和”为什么这么做”

    第三遍：查”公式”

    • 需要实现或深入理解时，才看详细公式
    • 公式是”代码”，理解是”注释”

    算法选择速查表

    问题类型 推荐算法 原因
    线性规划 单纯形法 / 内点法 多项式时间，工业标准
    小规模光滑无约束 牛顿法 / BFGS 收敛快，几十步搞定
    大规模无约束 L-BFGS / 共轭梯度 低内存，可扩展
    二次规划 共轭梯度 / 内点法 利用结构，n 步收敛
    Box 约束（$l \le x \le u$） 投影梯度 投影简单
    等式约束 梯度投影 / 拉格朗日乘子法 降维或投影
    一般非线性约束 SQP / 增广拉格朗日 工业级求解器
    不可导 / 非凸 启发式（遗传、模拟退火） 全局搜索

    最后的最后

    优化理论的美在于：

    1. 统一性：KT 条件统一了所有约束优化问题
    2. 层次性：从简单到复杂，每层都在前层基础上
    3. 实用性：支撑着机器学习、控制论、运筹学的核心算法

    记住：没有完美算法，只有合适的算法。

    理解原理，选对工具，调好参数，问题自然迎刃而解。
